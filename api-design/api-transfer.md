# データ転送 API (v1)

## ファイルダウンロード
指定されたパスにあるファイルをダウンロードします。

### リクエスト
- **URL:** `/v1/file`
- **Method:** `GET`

#### クエリパラメータ

| パラメータ名 | 必須 | 型 | フォーマット | デフォルト値 | 値の例 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `path` | 〇 | `string` | 1文字以上の文字列 | - | `"data/sample"` | `DATA_ROOT` からの相対パス<br>以下の場合にはエラーとなる<br>・空文字<br>・絶対パス (`/`で始まる)<br>・`../`を含む |


### レスポンス
- **200 OK:** ファイルバイナリを返却。
- **Error:** [共通エラー形式](./api-endpoint.md) に基づく JSON を返却。
  - `type` 値は以下参照

#### このエンドポイント固有のエラー識別子 (type)
`type` URLの末尾には以下の識別子が設定されます。

| 識別子 (type末尾) | ステータス | 理由 |
| :--- | :--- | :--- |
| `missing-parameter` | 400 | `path` 未指定 |
| `invalid-path-format` | 400 | トラバーサル等の不正な形式 |
| `out-of-bounds-access` | 403 | `DATA_ROOT` 範囲外へのアクセス |
| `file-not-found` | 404 | 指定ファイルが存在しない |

## ファイルアップロード
指定されたパスにあるファイルをアップロードします。

### リクエスト
- **URL:** `/v1/files`
- **Method:** `POST`

#### クエリパラメータ

| パラメータ名 | 必須 | 型 | フォーマット | デフォルト値 | 値の例 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `path` | 〇 | `string` | 1文字以上の文字列 | - | `"data/upload"` | `DATA_ROOT` からの相対パス<br>以下の場合にはエラーとなる<br>・空文字<br>・絶対パス (`/`で始まる)<br>・`../`を含む |

### リクエストボディ
`multipart/form-data` 形式で送信してください。

| パラメータ名 | 必須 | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| `files` | 〇 | `file` | 保存するファイルデータ。複数同時送信をサポート。 |

#### ファイル制限
- **最大サイズ:** 1GB / ファイル
- **禁止拡張子:** 以下のファイルは、サーバーのセキュリティ保護のため保存が拒否されます。
  - `"exe"`, `"msi"`, `"bat"`, `"cmd"`, `"ps1"`, `"vbs"`, `"sh"`, `"bin"`, `"app"`, `"jar"`, `"py"`, `"php"`, `"js"`
- **推奨拡張子:** 以下ファイルは、メディアファイルとして**警告なし**で保存されます。
  - **画像:** `"jpg"`, `"jpeg"`, `"png"`, `"gif"`, `"webp"`
  - **動画:** `"mp4"`, `"m4v"`, `"avi"`, `"wmv"`, `"mov"`, `"webm"`
  - **音声:** `"mp3"`, `"aac"`, `"wav"`, `"flac"`, `"alac"`
  - **Note:** リスト外の拡張子（例: `.zip`, `.pdf`）は、アップロード自体は可能ですが、レスポンスの `detail` に `WARNING` が記録されます。
- **ファイル名:** 以下`isValidName` の基準を満たさないもの
  - OS禁止文字(`< > : " | ? *`)を含まない
  - Windows予約名ではない
    - `"CON"`, `"PRN"`, `"AUX"`, `"NUL"`, `"COM1"`, `"COM2"`, `"COM3"`, `"COM4"`, `"COM5"`, `"COM6"`, `"COM7"`, `"COM8"`, `"COM9"`, `"LPT1"`, `"LPT2"`, `"LPT3"`, `"LPT4"`, `"LPT5"`, `"LPT6"`, `"LPT7"`, `"LPT8"`, `"LPT9"`
  - `.` で始まらない (隠しファイル)

### レスポンス
- **200 OK:**
  - `status`: `"success"`
  - `summary`: 処理結果の統計情報
  - `detail`: 警告（ホワイトリスト外の拡張子など）やエラーが発生したファイルの詳細リスト

- データクラス
  - `summary`
    - `total`: アップロードされたファイル数
    - `warning_count`: 警告が出た数

  - `detail`
    - `file`: ファイル名
    - `status`: 警告時のステータス
      - `WARNING`:ホワイトリスト外の拡張子で保存はされている
      - `ERROR`: 致命的なエラーで保存されなかった
      - `SKIPPED`: 致命的ではない理由でスキップされた
    - `reason`: 検知された具体的な理由

#### レスポンスボディ例
```
{
    "status": "success",
    "summary": {
        "total": 3,
        "warning_count": 1
    },
    "detail": [
        {
            "file": "sample",
            "status": "WARNING",
            "reason": "Non-whitelisted"
        },
        {
            "file": "copyLimit(100)",
            "status": "ERROR",
            "reason": "Duplicate limit exceeded"
        }
    ]
}
```

- **Error:** [共通エラー形式](./api-endpoint.md) に基づく JSON を返却。
  - `type` 値は以下参照

#### このエンドポイント固有のエラー識別子
`type`:  URLの末尾には以下の識別子が設定されます。

| 識別子 (type末尾) | ステータス | 理由 |
| :--- | :--- | :--- |
| `invalid-path-format` | 400 | リクエストされた path が不正な形式であった |
| `unsupported-file-name` | 400 | ファイル制限項目のファイル名に当てはまるファイル名の場合 |
| `unsupported-file-type` | 415 | 上記ブラックリストに該当するファイルが存在する場合 |
| `file-too-large` | 413 | 規定サイズを超過したファイルがリクエストされた |
| `no-files` | 400 | 指定ファイルが存在しない |
| `upload-failed` | 500 | サーバとの通信に障害が生じた |
